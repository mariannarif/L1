Basicos

Todos los comandos se ven así: "< _ >". El comando para finalizar es "< / _ >"
1. Se define tipo de documento con !DOCTYPE html
2. Backstage del documento: todo lo que define al doc (el nombre de la pagina web, por ejemplo) que no aparece en la página. Todo documento comienza con html, head, title, Nombre a aparecer en pestaña, /title, /head. Todo lo que esta entre los tags de head es la metadata del documento.
3. Comienza el body del documento.

Cada uno de estos es un elemento: html, body, h,p.

HTML elements with no content are called empty elements. < br > is an empty element without a closing tag (the < br > tag defines a line break). Empty elements can be "closed" in the opening tag like this: < br />. Ejemplo de < br >:

					!DOCTYPE html  
					html

					/body

Tags:

< h1 > < /h1 > titulo - hasta h6
< p > < /p > párrafo
< br > siguiente renglon (empty element-sin comando de cierre)
< hr > crea una linea divisoria (empty element)
< pre >< /pre > conserva el formato que se de en el editor de textos sin requerir p o br. La fuente dentro del pre es diferente a la predeterminada de p o h. Introducir h o p dentro de pre ignora el formato predeterminado.

Atributos

Sobre idiomas, links, imagenes,v ...
Proveen informacion adicional para la página
name="value".

Lang

Declara el idioma para bases de busqueda. Se define al inicio del documento, en html.
< html lang="en-US">

Title

Información adicional al texto. Se define en p.
title="blah"
Se recomienda siempre utilizar doble comilla para el valor del atributo; se pueden combinar comillas sencillas para titulos: title="Juan 'El salvador' Prz."

href

Para generar ligas. En el tag < a >. El link va entre comillas, seguido del texto a ligar.
< a href="liga"> Nombre < /a >
Link a atributos

Img

Nombre del archivo y tamaño:
< img src="wabi.png" width="104" height="142">

Alt

Provee informacion textual alternativa a la imagen para lector de textos
< img src="w3schools.jpg" alt="W3Schools.com" width="104" height="142">

Formato

 Se utilizan tags de formato para resaltar el texto visual y semanticamente. Requieren tag de cierre < /_ >.
	Visual: para el lector
		< b > - Bold text < /b >
		< i > - Italic text
		< mark > - Marked text
		< small > - Small text
		< del > - Deleted text
		< ins > - Inserted text
		< sub > - Subscript text
		< sup > - Superscript text
	Semantic: para el lector y metadata
		< strong > - Important text < /strong >
		< em > - Emphasized text
